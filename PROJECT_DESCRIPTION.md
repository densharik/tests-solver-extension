# MTI Solver — Описание проекта

## Что это

Расширение для Chrome, которое автоматически решает онлайн-тесты на платформе МТИ/Синергия (LMS) с помощью нейросетей (Claude и GPT). Пользователь устанавливает расширение, вводит токен доступа — и при открытии любого теста система сама читает вопрос, отправляет его AI, получает ответ и проставляет его в тесте.

## Как это работает (глазами пользователя)

1. **Установка**: пользователь ставит расширение в Chrome, открывает настройки панели, вводит свой токен доступа.
2. **Запуск теста**: при открытии страницы теста (`/assessments/...`) в правой или левой части экрана появляется плавающая панель «AI Решатель».
3. **Решение вопроса**: панель автоматически считывает текст вопроса и варианты ответов со страницы, отправляет на сервер, получает ответ AI и тут же кликает правильный вариант (или вписывает текст, перетаскивает элементы для сопоставления и т.д.).
4. **Автопилот**: можно нажать «Автопилот», указать количество прогонов — и система будет сама запускать тест, отвечать на все вопросы, сдавать тест, и повторять N раз подряд без участия пользователя.
5. **Итог**: после завершения теста панель показывает результат (X из Y правильных), сохраняет статистику и позволяет скопировать лог в буфер обмена.

## Типы вопросов, которые поддерживаются

- **Одиночный выбор (radio)** — выбрать один правильный вариант из нескольких
- **Множественный выбор (checkbox)** — выбрать несколько правильных вариантов
- **Свободный текст (text)** — вписать слово или фразу (обычно заполнение пропуска)
- **Сопоставление (matching)** — соединить элементы левого столбца с правым (drag-and-drop)
- **Группировка (matching_multi)** — распределить элементы по группам (каждой группе — несколько элементов)
- **Сортировка (sorting)** — расставить элементы в правильном порядке

## Модель заработка (SaaS)

Проект работает по кредитной модели:
- Администратор создаёт **токен** для клиента и начисляет ему **кредиты** (каждый кредит = одно решение вопроса).
- При каждом запросе к AI списывается 1 кредит. Если AI вернул ошибку — кредит возвращается.
- Клиент видит остаток кредитов в панели.
- Если ответ берётся из кэша (вопрос уже решался и ответ верифицирован) — кредит НЕ списывается.
- Токен может иметь срок действия (`expires_at`).

## Компоненты системы

### 1. Расширение Chrome (фронтенд)
Файл `content/content.js` — это весь клиентский код. Он:
- Инжектируется на страницы тестов LMS
- Рисует тёмную плавающую панель, которую можно двигать, сворачивать, прикреплять к тесту
- Парсит DOM страницы: определяет тип вопроса, извлекает текст и варианты
- Собирает изображения из вопроса (если есть картинки — отправляет их как base64)
- Отправляет промпт на бэкенд и получает JSON-ответ
- Применяет ответ: кликает кнопки, заполняет поля, перетаскивает элементы
- После теста парсит таблицу результатов и отправляет статистику

Настройки хранятся в `chrome.storage.local`:
- Токен, контекст (описание предмета), выбранная модель, дисциплина, позиция панели

### 2. Сервер (бэкенд) — Cloudflare Worker
Файл `worker_v5.js` — serverless функция на Cloudflare. Он:
- Принимает запрос с промптом и метаданными
- Проверяет токен, кредиты, срок действия, rate limit
- Проверяет кэш: если этот же вопрос уже решался и ответ верифицирован как правильный — возвращает из кэша бесплатно
- Если кэша нет — отправляет промпт в Claude API (Anthropic) или OpenAI API в зависимости от выбранной модели
- Может включить web search (инструмент `web_search_20250305` у Claude) для поиска ответа в интернете
- Списывает кредит, логирует запрос, возвращает ответ

### 3. База данных — Cloudflare D1 (SQLite)
Таблицы:
- `tokens` — токены клиентов с балансом кредитов и сроком действия
- `usage_log` — лог всех запросов (кто, когда, какой вопрос, какая модель)
- `question_cache` — кэш вопросов и ответов с верификацией
- `test_results` — итоги тестов (сколько правильных из скольких)

## Ключевые фичи

### Кэш с верификацией
Когда пользователь решает тест, после получения результатов расширение сверяет каждый вопрос с тем, что ответил AI, и отправляет на сервер батч-верификацию: для каждого вопроса — правильно или нет. Если ответ оказался верным — он верифицируется в кэше. В будущем другие пользователи получат этот ответ бесплатно из кэша.

### Выбор модели
Пользователь может выбрать:
- **Claude Sonnet** — основная модель, высокая точность
- **Claude Haiku** — быстрая и дешёвая
- **GPT mini / GPT nano** — модели OpenAI

### Дисциплины
Для каждой дисциплины формируется свой системный промпт с ролью эксперта:
- IT / Технические
- Право
- Экономика / Менеджмент / HR
- Психология
- Математика / Науки

### Компетентностные тесты
Отдельная логика промптов для тестов-кейсов: AI инструктируется анализировать конкретные данные из условия, а не полагаться на общие знания.

### Автопилот
Полностью автоматический режим: запуск теста → ответ на каждый вопрос → нажатие «Ответить» → когда все вопросы пройдены — «Сдать тест» → ожидание результатов → запуск следующего прогона. С watchdog-таймером на случай зависаний.

### Логирование
После каждого теста формируется строка лога в TSV-формате (для вставки в Google Sheets), содержащая:
- Количество вопросов и правильных ответов
- Дисциплина, модель, длительность
- Подробности по каждому вопросу (из попапа статистики LMS)
- Флаги: промежуточный тест, компетентностный, с обоснованием

### Web Search
Кнопка «Повторить с веб-поиском» — AI ищет ответ в интернете через встроенный инструмент web search, что повышает точность на фактологических вопросах.

## Доступные модели AI

| Модель | Провайдер | Характеристика |
|--------|-----------|----------------|
| Claude Sonnet 4.6 | Anthropic | Основная, высокая точность |
| Claude Haiku 4.5 | Anthropic | Быстрая, дешёвая |
| GPT 5.4 mini | OpenAI | Средняя |
| GPT 5.4 nano | OpenAI | Самая дешёвая |

## Администрирование

Через скрипт `admin.sh` или напрямую через API (`/admin/*` эндпоинты с ключом `ADMIN_KEY`):
- Создание токенов с начальным балансом
- Пополнение кредитов
- Просмотр списка токенов и статистики
- Удаление токенов

## Деплой

- Worker деплоится через `wrangler deploy`
- Расширение устанавливается в Chrome через режим разработчика (Load unpacked)
- Секреты (API-ключи) устанавливаются через `wrangler secret put`
